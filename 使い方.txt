================================================================================
  FullGC 解析ツール  使い方ガイド  v2.0
  対象システム: Java 8 + Spring Boot 1.3 + Tomcat
================================================================================

--------------------------------------------------------------------------------
1. 起動方法
--------------------------------------------------------------------------------

  gc-analyzer.html をダブルクリックしてブラウザで開くだけで起動します。
  インストール・コンパイル不要。インターネット接続不要。

  動作確認済みブラウザ: Chrome / Edge / Firefox


--------------------------------------------------------------------------------
2. ログの準備
--------------------------------------------------------------------------------

  各サーバーから以下の 5 種類のログを取得して、ローカルに保存してください。
  すべて揃わなくても、読み込んだ分だけ解析します。

  ┌──────────────┬──────────────────────────────────────────────────────────────┐
  │ ログ種別     │ 取得コマンド例                                               │
  ├──────────────┼──────────────────────────────────────────────────────────────┤
  │ jstat-gc     │ jstat -gc -t <PID> 1000 >> jstat_gc.log                     │
  │              │  ※ -t オプション推奨（時系列相関が可能になります）          │
  │              │  ※ FullGC 多発時間帯を含む期間を取得してください            │
  ├──────────────┼──────────────────────────────────────────────────────────────┤
  │ jstack       │ jstack <PID> >> jstack.log                                   │
  │ (スレッドダンプ)│  ※ FullGC 多発中に複数回取得すると精度が上がります      │
  ├──────────────┼──────────────────────────────────────────────────────────────┤
  │ jmap-heap    │ jmap -heap <PID> > jmap_heap.log                             │
  │ (ヒープ構成) │  ※ FullGC 発生前後に取得するとヒープ状態を把握できます     │
  ├──────────────┼──────────────────────────────────────────────────────────────┤
  │ jmap-histo   │ jmap -histo:live <PID> > jmap_histo.log                     │
  │ (オブジェクト)│  ※ live オプションを付けると GC 後の生存オブジェクトのみ  │
  │              │     対象になり、実際のメモリ占有状況が分かります             │
  ├──────────────┼──────────────────────────────────────────────────────────────┤
  │ アクセスログ  │ Tomcat の logs/localhost_access_log.*.txt                   │
  │              │  ※ FullGC 多発時間帯を含む日付のファイルを使用              │
  └──────────────┴──────────────────────────────────────────────────────────────┘

  【推奨】アクセスログにレスポンスタイムを出力する設定（server.xml）:
    <Valve className="org.apache.catalina.valves.AccessLogValve"
           pattern="%h %l %u %t &quot;%r&quot; %s %b &quot;%{Referer}i&quot; &quot;%{User-Agent}i&quot; %D" />
    ※ %D = レスポンスタイム（マイクロ秒）。これがないと遅延分析ができません。

  【PID の調べ方】
    jps -l          → Java プロセス一覧から Spring Boot アプリの PID を確認
    または
    ps -ef | grep java  (Linux の場合)


--------------------------------------------------------------------------------
3. 解析手順
--------------------------------------------------------------------------------

  手順 1: ログファイルを読み込む
    ・画面上の 6 つのサーバーパネルが表示されます
    ・各サーバーのパネルで「選択」ボタンをクリックし、ログファイルを選択
    ・ログが 5 種類すべてなくても、読み込んだ分だけ解析します
    ・サーバー名は入力欄で変更できます（例: ap01, web-server-1 など）

  手順 2: 解析実行
    ・「▶ 解析実行」ボタンをクリック
    ・読み込んだサーバー分のタブが生成されます

  手順 3: レポートを確認
    ・タブをクリックして各サーバーのレポートを切り替えます
    ・「📊 比較サマリ」タブで全サーバーの横並び比較が確認できます

  手順 4: レポートを保存
    ・「↓ レポート保存 (.txt)」ボタンで全レポートをまとめてテキスト保存
    ・「クリップボードにコピー」ボタンで表示中のレポートをコピー


--------------------------------------------------------------------------------
4. 出力レポートの見方
--------------------------------------------------------------------------------

  【サーバー個別レポート】

  [1] FULL GC サマリ
    ・FullGC 回数・合計時間・平均/最大所要時間を表示
    ・イベント一覧に「行/経過時刻」「GC 直前の OldGen 使用率」を表示
    ・OldGen 90% 超で「(**)」マークが付きます
    ・GC オーバーヘッドが 10% 超の場合は WARNING を表示
    ・jstat に -t オプションがない場合は行番号で代替します

  [2] OLD GEN 使用率推移
    ・ASCII グラフで使用率の変化を視覚化
    ・85% を超えた行に「← 警告」が表示されます
    ・OldGen 容量 (MB) も確認できます

  [3] YOUNG GC サマリ
    ・Young GC 回数・合計時間・平均時間を表示
    ・Young GC が頻発している場合は Promotion（Old 昇格）の増加を疑います

  [4] スレッドダンプ解析  ← jstack ログを使用
    ・全ダンプのスレッド状態分布（平均値）を表示
    ・BLOCKED スレッドが 5 超で WARNING を表示
    ・BLOCKED スパイク発生時刻とスレッド詳細を表示
    ・頻出スタックフレーム Top5 でボトルネックの手がかりを確認

  [5] ヒープダンプ解析  ← jmap ログを使用 ★v2.0 新機能
    ・jmap -heap: ヒープ構成（MaxHeapSize / NewRatio / SurvivorRatio）を表示
    ・jmap -heap: 各領域（Eden / Survivor / OldGen / Metaspace）の使用率を
                 ASCII グラフで表示。80% 超で警告が出ます
    ・jmap -histo: オブジェクト Top10（インスタンス数・サイズ MB・割合）を表示
    ・jmap -histo: アプリ固有クラス（JDK 標準以外）を自動検出して別掲します
                 → メモリを大量消費しているアプリコードの特定に役立ちます

  [6] アクセスログ解析
    ・総リクエスト数・遅延リクエスト数（3 秒超）を表示
    ・ステータスコード分布（2xx/3xx/4xx/5xx）
    ・リクエスト集中時間帯 Top5（分単位）
    ・URI 別 Top10（件数・平均/P95/最大レスポンスタイム）
    ・平均 3 秒超の URI に「★遅延」マークが付きます

  [7] 推奨事項
    ・[HIGH] / [MEDIUM] / [INFO] の優先度で改善提案を表示
    ・ログの読み込み不足がある場合も案内します

  【比較サマリ】
    ・6 台の主要メトリクスを横並びで比較
    ・FullGC 回数の外れ値サーバーを自動検出
    ・OldGen 使用率（jmap）・ヒープ使用量 MB（histo）を横並び比較 ★v2.0 追加
    ・遅延 URI の全サーバー出現状況を比較
    ・全体の推奨事項を優先度付きで表示


--------------------------------------------------------------------------------
5. FullGC 原因特定のポイント
--------------------------------------------------------------------------------

  ■ OldGen 使用率が 85〜90% で FullGC が発生している場合
    → ヒープサイズが不足している可能性
    → [5] ヒープダンプ解析の「ヒープ構成」で MaxHeapSize を確認
    → 対処: -Xmx の値を増やす

  ■ jmap -histo でアプリ固有クラスのサイズが大きい場合
    → そのクラスのインスタンスが解放されずに Old Gen に蓄積している可能性
    → 対処: 当該クラスのライフサイクル・キャッシュ設計を見直す

  ■ 特定の URI の直後に FullGC が発生している場合
    → そのリクエストが大量のオブジェクトを生成している可能性
    → 対処: 当該処理のページネーション・バッチ分割・キャッシュ見直し

  ■ BLOCKED スレッドが多い場合
    → DB 接続プールの枯渇やロック競合の可能性
    → [4] スレッドダンプの「頻出スタックフレーム」でボトルネックを特定
    → 対処: 接続プールサイズ・タイムアウト設定の見直し

  ■ 月初だけ FullGC が多発する場合
    → 月次バッチ処理・月次集計レポート・大量データ読み込みが疑われます
    → [6] アクセスログの「リクエスト集中時間帯」と FullGC 発生時刻を照合
    → [5] ヒープダンプで月初処理後のオブジェクト残留状況を確認

  ■ サーバー間でバラつきがある場合
    → 比較サマリの「外れ値検出」で高負荷サーバーを確認
    → ロードバランサーの振り分けやバッチ割当を確認

6. よくある質問
--------------------------------------------------------------------------------

  Q. ログを読み込んでも解析結果が空になる
  A. ファイルの文字コードが UTF-8 以外の場合に発生することがあります。
     メモ帳などで UTF-8 形式で保存し直してからお試しください。

  Q. jstat のタイムスタンプがすべて N/A になる
  A. jstat 取得時に -t オプションが付いていない場合です。
     次回から「jstat -gc -t <PID> 1000」で取得してください。

  Q. アクセスログの平均レスポンスタイムがすべて「-」になる
  A. Tomcat の AccessLogValve に %D または %T が含まれていないためです。
     上記「2. ログの準備」の推奨設定を参照してください。

  Q. jmap -histo の「アプリ固有クラス」に何も表示されない
  A. 全クラスのサイズが合計の 0.5% 未満の場合は表示されません。
     Top10 欄でクラス名を直接確認してください。

  Q. jmap -heap の使用率が全部 N/A になる
  A. GC 種別（SerialGC / ParallelGC / G1GC）によって出力フォーマットが
     異なります。G1GC の場合は出力形式が大きく異なるため、
     jmap -histo のみをお使いください。

  Q. 6 台以上のサーバーを解析したい
  A. 現バージョンは 6 台固定です。複数回に分けてご利用ください。
